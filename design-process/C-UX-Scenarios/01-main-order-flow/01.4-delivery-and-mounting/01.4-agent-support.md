# 01.4 Agent Support — Delivery & Mounting / Guided Checkout

**Companion to:** [01.4-Delivery and Mounting](./01.4-delivery-and-mounting.md)
**Scenario:** 02 — Man Purchasing Flow, Path A (Assisted Purchase)
**Agent role at this step:** Form Filler, Step Navigator

---

## What the Agent Does Here

The checkout panel is open. The agent offers to prefill each checkout step in sequence — delivery method, then address or workshop details. The user can accept, edit, or skip the agent at any point.

This is the most form-heavy part of the flow. The agent's value here is concrete: fewer taps, faster completion.

---

## Step 1 — Delivery Method (Leveringsmåte)

The agent does **not** select the delivery method. This choice is too consequential — workshop vs. home delivery affects price, timing, and the booking step. The user taps.

The agent can clarify options if asked:

> *Bruker:* Hva er forskjellen på montering og hjemlevering?
> *Agent:* **Montering:** Vi sender dekkene til et verksted, og du bestiller time for montering der. Inkludert i prisen. **Hjemlevering:** Vi sender dekkene til din adresse. Du monterer selv eller avtaler med eget verksted.

After the user selects a method and taps "Neste":

> *Agent:* Vil du at jeg fyller inn adressen din?

---

## Step 2 — Address / Workshop Details (Kundeopplysninger / Leveringsadresse)

If the user accepts, the agent prefills each field sequentially:

1. First name → amber pulse → settles
2. Last name → amber pulse → settles
3. Address line → amber pulse → settles
4. Postal code → amber pulse → settles (city autofills)
5. Email → amber pulse → settles
6. Phone → amber pulse → settles

Each field follows the [Field Fill Interaction Pattern](../../02-man-purchasing-flow/02-man-purchasing-flow.md#agent-field-fill-interaction-pattern-path-a).

The agent only prefills if it has the data. In a cold session (no prior order data), it asks:

> *Agent:* Hva er navn og adresse? *(user types or speaks)* → agent parses and fills

If the user has placed an order before and authenticated via Path B, the agent may have address data from a prior order — it fills from that.

After all fields filled:
> *Agent:* Ser det bra ut? Trykk **Neste** når du er klar, så går vi videre til betaling.

The user reviews, edits if needed, and taps the step's "Neste" button (not the agent — step navigation is always user-triggered at this step).
