# 01.2 Agent Support — Product Cards

**Companion to:** [01.2-Product Cards](./01.2-product-cards.md)
**Scenario:** 02 — Man Purchasing Flow, Path A (Assisted Purchase)
**Agent role at this step:** Qualifier, Advisor, Recommender, Selector

---

## What the Agent Does Here

The user is on the results screen looking at matching tires. They may feel paralyzed by choice, or they may not know that a 30 kr/month difference in fuel cost over 4 years adds up to more than the price difference between a B and an A-rated tire. The agent's job is to ask the right questions, understand what actually matters to this person, and commit to one recommendation with a clear reason.

This is also the natural upgrade moment. The agent does not push — but if the user cares about silence, the environment, or comfort, there is almost certainly a better tire than the cheapest one. The agent surfaces that.

---

## Qualification Flow

The agent asks short, conversational questions — never more than 3 before committing to a recommendation. Each question has a direct mapping to EU label priorities or product tier.

**The agent is passive by default.** It does not interrupt the user when results load. The qualification flow only starts when the user opens the chat and asks for help — "hva skal jeg velge?", "vet ikke hvilken", "kan du hjelpe meg?" or similar.

If the user browses and selects a tire independently, the agent stays silent. It activates in checkout if needed.

When the user does ask:
> *Agent:* Jeg kan hjelpe deg velge rett dekk. Har du 30 sekunder?

---

### Question 1 — What matters beyond price?

> *Agent:* Hva er viktigst for deg, bortsett fra pris?

This is deliberately open. The user's answer maps to one or more priority signals:

| User says | Agent interprets as | Primary EU label signal |
|-----------|--------------------|-----------------------|
| Sikkerhet / grep / bremsing | Safety-first | Wet grip (A) |
| Stille / musikk / lyd / komfort | Comfort/silence | Noise (lowest dB) |
| Miljø / drivstoff / forbruk / klima | Eco/efficiency | Fuel efficiency (A) |
| Lang levetid / holdbar / slitasje | Longevity | Combined score + tread wear rating |
| Vet ikke / ingen preferanse / bare bytt | No preference | Default: best combined score |

If the user names multiple priorities (e.g. "stille og godt grep"), the agent weights both and finds the best overlap.

---

### Question 2 — What kind of trips?

> *Agent:* Kjører du mest i byen, eller er det mye landevei og motorvei?

| User says | Agent interprets as | Effect on recommendation |
|-----------|--------------------|-----------------------|
| By / kort / jobb / handling | Urban/short | Fuel efficiency matters more (stop-start) |
| Landevei / motorvei / tur / pendler | Highway/long | Wet grip and noise matter more |
| Begge deler / variert | Mixed | Balanced — best combined score |

---

### Question 3 — Silence check (only if not already answered)

Asked only if the user hasn't mentioned sound/comfort in Q1:

> *Agent:* Hører du mye på musikk eller podcast når du kjører?

| User says | Effect |
|-----------|--------|
| Ja / alltid / mye | Add noise rating as secondary filter — recommend tires ≤ 70 dB |
| Nei / sjelden | Noise is not a factor |

---

### Upgrade moment

After Q1–Q2 (and Q3 if asked), if the cheapest result is not the best match, the agent surfaces the gap honestly:

> *Agent:* Det billigste dekket koster 649 kr, men basert på det du sa om stille kjøring anbefaler jeg Michelin Primacy 4 til 849 kr — den er 5 dB roligere og har samme veigrep. Det er 200 kr mer per dekk, men over 40 000 km merkes det tydelig.

The agent names the difference, explains the benefit in practical terms, and lets the user decide. It does not repeat the pitch if the user declines.

---

## Recommendation Commit

After the qualifying questions, the agent commits to exactly one tire. It does not offer two options or say "it depends." If the user wants to compare, they can ask.

> *Agent:* Basert på det du fortalte meg anbefaler jeg **[Merke] [Modell]**. [One-sentence reason tied to their stated priority.]

Then the inline product card renders.

---

## Inline Product Card

The agent renders the recommended tire as a compact inline card:

```
┌─────────────────────────────┐
│ [tire image]                │
│ Michelin Primacy 4          │
│ 205/55R16 91V               │
│ Drivstoff: A  Grep: A  Støy: 68dB │
│ 849 kr / stk                │
│ ✓ På lager                  │
│                             │
│ [Velg denne]                │
└─────────────────────────────┘
```

The card shows price, EU ratings, and stock status. Tapping **Velg denne** or typing "ja" / "den" / "ok" selects it.

---

## Agent-Assisted Selection

When the user confirms:

1. Agent calls `handleSelectTire` — same as tapping **Velg disse** on the card
2. The "Velg disse" button on the card goes green for 400ms
3. Checkout panel opens
4. Agent: *"Bra valg! Jeg åpner kassen."*

The user can also tap any card directly, bypassing the recommendation. The agent continues in checkout regardless.

---

## EU Label Explanation

If asked:

> *Agent:* EU-merket har tre vurderinger:
> - **Drivstoff (A–E):** Hvor mye dekket øker bensinforbruket. A er best — kan utgjøre hundrevis av kroner i året.
> - **Veigrep (A–E):** Bremseavstand på våt vei. A betyr kortere bremsevei — den viktigste faktoren for sikkerhet.
> - **Støy (dB):** Lydnivå inne i bilen. Under 70 dB merkes tydelig om du hører på musikk.

## Agent Capabilities at This Step

| Capability | Allowed | Notes |
|-----------|---------|-------|
| Read current view | ✓ | Knows user is on results |
| Read visible products | ✓ | EU labels, prices, noise ratings, stock |
| Ask qualifying questions | ✓ | Max 3 before committing |
| Surface upgrade recommendation | ✓ | Named price diff + practical benefit |
| Show inline product card | ✓ | Single recommended tire |
| Select a tire via confirmation | ✓ | Calls `handleSelectTire` |
| Compare two tires side-by-side | ✓ | If user asks "hva er forskjellen?" |
| Open product detail view | ✓ | If user wants more info first |
| Apply filters on behalf of user | ✗ | User controls filters |
