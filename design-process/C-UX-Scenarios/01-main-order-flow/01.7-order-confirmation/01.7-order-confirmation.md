## Overview

**Page Purpose:** Close the loop on the purchase. Show the customer everything they ordered, confirm the booked mounting time, invite them to rate their experience, and open a direct line to AI support — all in one scrollable view.

**User Situation:** Harriet has completed payment and (optionally) booked a mounting time. The checkout panel transitions to the confirmation view. No back navigation is available — the order is placed.

**Success Criteria:**
- Harriet sees her complete order at a glance
- She can verify and change her mounting time if needed
- She rates her experience (optional but prompted)
- She can ask a question or report an issue via AI chat without calling support

**Entry Points:**
- Order placed successfully from CheckoutPanelContent (`onSuccess` callback)

**Exit Points:**
- Menu button → home (order complete, no back)
- "Ändra din tid" button → reopens booking step
- No other exits — this is the terminal state

## AI Behavior

The AI is Claude Haiku acting as a Sharif customer service agent.

**System prompt context passed on each request:**
- Order ID, product name, dimension, quantity, price
- Booked mounting time and shop
- Customer email

**Capable of handling:**
- Wrong tire dimension ordered → offer to initiate exchange
- Cancellation requests → explain process, offer alternatives
- Mounting time questions → confirm details, link to rebooking
- General product questions → answer from product data
- Anything else → warm handoff to human support (phone number)

**Tone:** Friendly, concise. Norwegian by default, switches to English if user writes in English.

**Data persistence:** All messages stored in `dialog_messages` linked to `dialog.id` → `order_id`. Accessible to support agents via admin view.

## Dialog Data Model

```
dialog
├── id
├── cart_id          (set when cart created)
├── order_id         (set when order placed)
├── customer_id      (if logged in)
├── customer_email
├── product_handle
├── dimension
├── outcome          purchased | abandoned | support
├── ai_summary       (generated on abandon)
├── created_at
├── converted_at
└── abandoned_at

dialog_messages
├── id
├── dialog_id
├── role             user | assistant | agent
├── content
└── created_at
```

**Abandoned cart value:** Dialogs without `order_id` (cart abandoned) are preserved. Admin view can filter on `outcome = 'abandoned'` + `dialog_messages IS NOT EMPTY` to see exactly what questions caused customers to leave.

## Technical Notes

- The confirmation view is a FlowShell panel state (`view === "confirmation"`), not a separate page/route
- `onSuccess(orderId)` callback from CheckoutPanelContent triggers the view switch in FlowShell
- Header shows menu button (not back arrow) — `view === "confirmation"` treated same as `"home"` for header state
- Dialog is created when cart is created, updated with `order_id` and `converted_at` when order is placed
- Star rating can be stored in the `dialog` table as `rating: int` (add column in migration)
- "Ändra din tid" reopens the booking sub-step — implementation TBD (may require reopening checkout panel at booking step)
- AI reply is shown inline below the textarea (not in a separate chat view) — keeps the page simple
