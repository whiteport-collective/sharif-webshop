# 01.5 Agent Support — Payment

**Companion to:** [01.5-Payment](./01.5-payment.md)
**Scenario:** 02 — Man Purchasing Flow, Path A (Assisted Purchase)
**Agent role at this step:** Passive — Handover

---

## What the Agent Does Here

Payment is a hard boundary. The agent does not touch, suggest, or fill any payment fields. It does not read card numbers or payment data.

When the user reaches the payment step, the agent hands over explicitly and minimizes.

---

## Handover Message

> *Agent:* Bra jobbet! Nå er det bare betaling igjen — det tar du selv. Lykke til med de nye dekkene! 🛞

The agent panel minimizes (collapses to the header icon). The Stripe payment form loads and takes full focus.

---

## If the User Asks the Agent Something During Payment

The agent is still accessible via the chat icon. It can answer non-payment questions:

| Question | Agent response |
|----------|---------------|
| "Hva koster frakten?" | Reads from cart summary — answers with shipping cost |
| "Hva er returpolicyen?" | Answers from knowledge base |
| "Hva skjer etter jeg betaler?" | Explains: order confirmation email, then booking step |
| "Noe gikk galt med kortet" | *"Prøv igjen, eller ring oss på +47 934 85 790 om problemet vedvarer."* |

The agent does **not**:
- Attempt to re-enter payment details
- Read or repeat back card numbers, CVV, or expiry
- Suggest workarounds to payment failures

---

## Agent Capabilities at This Step

| Capability | Allowed | Notes |
|-----------|---------|-------|
| Answer general questions | ✓ | Shipping, returns, what happens next |
| Enter payment details | ✗ | Hard boundary |
| Read payment form content | ✗ | Hard boundary |
| Accept terms / submit order | ✗ | Hard boundary |

---

## Entry to Next Step

After successful payment, the order is placed. If the user selected workshop mounting, the app advances to [01.6 Book Mounting](../01.6-book-mounting/01.6-agent-support.md).
