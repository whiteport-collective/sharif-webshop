# 01.1 Agent Support — Dimension Input

**Companion to:** [01.1-Dimension Input](./01.1-dimension-input.md)
**Scenario:** 02 — Man Purchasing Flow, Path A (Assisted Purchase)
**Agent role at this step:** Guide, Interpreter, Field Filler

---

## What the Agent Does Here

The user is on the home screen. They may not know their tire size, or they may know it but not how to enter it. The agent bridges the gap between what the user knows and what the form needs.

Three situations the agent handles:

### Situation A — User types a dimension string in chat
User types "2055516" or "205/55R16" or "205 55 16".

Agent parses the input, fills the three form fields (width → profile → rim) sequentially with highlight animation, confirms the fill, then offers to trigger the search.

**Dialog:**
> *Bruker:* 2055516
> *Agent:* Jeg fyller inn dimensjonene dine — 205/55R16. Er det riktig?
> *(Fields fill with amber pulse)*
> *Bruker:* ja
> *Agent:* Bra! Jeg søker nå etter dekk som passer.
> *(Agent triggers "Finn dekk")*

---

### Situation B — User doesn't know their tire size
User opens the agent and types "vet ikke størrelsen" or "hvor finner jeg det?" or similar.

Agent explains the three places to look, in plain language:

1. **Side of the tire** — look for a number like 205/55R16 printed on the sidewall
2. **Door frame sticker** — open the driver's door, look for a white label on the door jamb — it lists the recommended tire size for the car
3. **Owner's manual** — under "Tires" or "Wheels"

Agent then waits for the user to report back, and proceeds with Situation A.

**Dialog:**
> *Bruker:* Vet ikke størrelsen min
> *Agent:* Ingen problem! Du finner den på tre steder:
> 1. På siden av dekket — se etter tall som "205/55R16"
> 2. Klistremerket inni dørkarmen på førersiden
> 3. I kjøretøyets manual under "Dekk"
> 
> Har du funnet noe?

---

### Situation C — User knows make/model but not size
User types "har en Golf 2019" or "Toyota Yaris 2021".

Agent responds that it currently works by dimension input (not car lookup), explains how to find the dimension, and guides toward Situation B.

> *Agent:* Jeg finner dekk basert på dekkdimensjon, ikke bilmodell ennå. Men det er enkelt å finne størrelsen — se på klistremerket inni dørkarmen. Hva står der?

---

## Field Fill Interaction Pattern

Follows the global pattern defined in [Scenario 02](../../02-man-purchasing-flow/02-man-purchasing-flow.md#agent-field-fill-interaction-pattern-path-a).

| Step | What happens |
|------|-------------|
| 1 | Agent announces: *"Jeg fyller inn dimensjonene dine"* |
| 2 | Width fills first — `ring-amber-400 animate-pulse` for ~800ms |
| 3 | Width settles to `bg-amber-50 ring-amber-200` (agent-filled state) |
| 4 | Profile fills next (same animation) |
| 5 | Rim fills last (same animation) |
| 6 | All three settled — agent confirms in chat, offers to proceed |
| 7 | User confirms or taps **Bekreft og fortsett** in chat |
| 8 | Agent triggers "Finn dekk" — parallax to 01.2 |

The user can tap any field to edit it before confirming. The amber highlight fades on tap or edit.

## Scope — What the Agent Will Not Answer

The agent answers questions about tires, wheels, road safety, and the Sharif ordering flow. Everything else is deflected in one line.

> *Bruker:* Kan du gi meg en pannekakeoppskrift?
> *Agent:* Det er utenfor det jeg kan hjelpe med — jeg er spesialist på dekk! Er det noe dekkrelatert jeg kan hjelpe deg med?

This applies at all steps. The deflection is short, friendly, and immediately redirects to the agent's domain.
