## What this screen does

Configuration hub for the Sharif admin extension. Two main sections: **Agent** (API key + instructions) and **Store** (operational defaults like low stock threshold and mounting locations).

This is where Moohsen — or whoever manages the system — connects the AI, customises its behaviour, and sets store-specific defaults. No code changes required.

---

## User actions

| Action | Result |
|--------|--------|
| Enter Anthropic API key + save | Encrypted and saved to `sharif_settings`, activates agent |
| Clear API key | Agent deactivated, sidebar shows "No key configured" |
| Edit agent instructions overlay + save | Custom rules merged on top of base prompt at runtime |
| View base instructions (read-only) | Collapsible — shows the built-in system prompt |
| Edit low stock threshold + save | Updates threshold used in 03.2 and 03.4 |
| Edit mounting locations | Add/remove/rename shop locations |

---

## Settings sections

### Section 1 — Agent

| Field | Type | Description | Stored in |
|-------|------|-------------|-----------|
| Anthropic API key | Password input (masked) | `sk-ant-...` key from console.anthropic.com | `sharif_settings.anthropic_api_key` |
| Key status | Read-only badge | ✓ Configured (green) / ⚠ Not set (amber) | Derived |
| Agent instructions overlay | Textarea, 6 rows | Custom rules added on top of base prompt | `sharif_settings.agent_instructions_overlay` |
| Base instructions | Collapsible, read-only | Shows the built-in system prompt from repo | `backend/src/admin/agent-instructions.md` |
| Agent model | Read-only | `claude-haiku-4-5-20251001` (Phase 1) | Hard-coded |

### Section 2 — Store defaults

| Field | Type | Description | Stored in |
|-------|------|-------------|-----------|
| Low stock threshold | Number input | Units per variant below which "Low stock" triggers | `sharif_settings.low_stock_threshold` |
| Mounting locations | List (add/remove) | Shop names used in order filtering + agent replies | `sharif_settings.mounting_locations` |
| Store name | Text input | Used in agent system prompt greeting | `sharif_settings.store_name` |
| Admin display name | Text input | Used in "God morgen, [name]" greeting | `sharif_settings.admin_name` |
| Default language | Select: no / en | Agent response language default | `sharif_settings.default_language` |

## Data connections

| Table / API | Purpose |
|-------------|---------|
| `sharif_settings` (local Postgres) | All configurable values — key-value store |
| `backend/src/admin/agent-instructions.md` | Base system prompt (read-only, from repo) |
| Medusa admin auth | Only authenticated admins can access Settings |

### `sharif_settings` schema

```sql
CREATE TABLE IF NOT EXISTS sharif_settings (
  id TEXT PRIMARY KEY,       -- e.g. 'anthropic_api_key', 'low_stock_threshold'
  value TEXT,
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

---

## Security note

The Anthropic API key is stored in plain text in local Postgres (no external service). This is acceptable for a single-operator local deployment. For a cloud deployment, use environment variables instead and hide this field.

## User Scenarios

### Scenario A — First-time setup: activating the agent

The system is freshly installed. The agent sidebar shows: "Jeg er ikke aktivert ennå. Legg inn en Anthropic API-nøkkel i Innstillinger for å komme i gang." Moohsen opens Settings, finds the API key field, pastes in the key, and clicks Save. The badge switches from amber "Ikke konfigurert" to green "Konfigurert." He navigates back to the dashboard and the agent greets him for the first time. He types "Hei" — the agent responds with the morning briefing. He is satisfied.

### Scenario B — Raising the low stock threshold after a bad weekend

Two weeks ago the system showed green on a product but Moohsen had only 4 units left — they sold over a single weekend. He opens Settings, finds the low stock threshold set to 4, changes it to 8, and saves. He also adds "Drammen Syd" to the mounting locations list — the shop has opened a second location and it needs to appear in the order filters and agent replies.

### Scenario C — Customising agent behaviour

Moohsen has noticed the agent sometimes suggests products that are not in stock when handling escalated customer questions. He opens Settings, scrolls to the agent instructions overlay, and types: "Foreslå aldri produkter som ikke er på lager. Hvis kunden spør etter et produkt vi ikke har, tilby nærmeste tilgjengelige alternativ og oppgi leveringstid." He saves. From the next session onward, the agent applies this rule on top of its base prompt. No code changes required.
