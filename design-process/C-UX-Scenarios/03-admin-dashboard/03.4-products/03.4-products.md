## What this screen does

Product catalogue and stock management in one view. Moohsen sees all tire products with current stock levels per variant (dimension). Low stock items are surfaced at the top with an amber/red indicator. He can filter by brand, season, and stock status, and ask the agent natural language inventory questions.

Phase 1: read-only + stock level editing.
Phase 2: full product management (create, update, bulk import from Excel — the Moohsen demo scenario).

---

## User actions

| Action | Result |
|--------|--------|
| Search by name / brand / dimension | Filters product list in real time |
| Filter by season | All / Summer / Winter studless / Winter studded |
| Filter by brand | Dropdown from available brands |
| Filter by stock status | All / In stock / Low stock / Out of stock |
| Click product row | Opens product detail panel (right column) |
| Edit stock quantity (inline) | Updates `InventoryLevel.stocked_quantity` |
| Ask agent "what's low stock?" | Agent calls `getLowStockProducts(threshold)` |

## Product detail panel (right column)

| Section | Fields | Source |
|---------|--------|--------|
| Header | Product name, brand, season badge | `Product` |
| Image | Thumbnail | `Product.images` |
| EU Labels | Fuel efficiency, wet grip, noise | `Product.metadata.eu_*` |
| Variants | Table: dimension, price, stock, status | `ProductVariant` + `InventoryItem` + `InventoryLevel` |
| Stock editing | Inline editable quantity per variant | `InventoryLevel.stocked_quantity` |
| Description | Product description | `Product.description` |

---

## Stock status thresholds

| Status | Condition | Colour |
|--------|-----------|--------|
| In stock | `stocked_quantity > threshold` | Green |
| Low stock | `0 < stocked_quantity ≤ threshold` | Amber |
| Out of stock | `stocked_quantity === 0` | Red |

Threshold is configurable in 03.7 Settings (default: 4 units per variant).

---

## Data connections

| Table / API | Purpose |
|-------------|---------|
| Medusa `Product` | Name, description, thumbnail, metadata |
| Medusa `ProductVariant` | Dimension (option value), price, SKU |
| Medusa `InventoryItem` | Linked to variant via `InventoryItem.sku` |
| Medusa `InventoryLevel` | `stocked_quantity`, `reserved_quantity` per location |
| Medusa `ProductCollection` | Brand grouping |
| `Product.metadata` | `season`, `eu_fuel`, `eu_grip`, `eu_noise` |
| `sharif_settings` | Low stock threshold |

## Agent behaviour on this screen

Context injected: `"User switched to Products & Inventory view."`

Example queries:

| Query | Agent action |
|-------|-------------|
| "Hvilke produkter er snart utsolgt?" | Calls `getLowStockProducts(threshold)` |
| "Vis alle sommerdekk fra Michelin" | Calls `listProducts({ season: "summer", brand: "Michelin" })` |
| "Hvor mange 205/55R16 har vi igjen?" | Calls `listProducts({ dimension: "205/55R16" })`, sums stock |
| "Oppdater lager for Nokian Tyres 205/55R16 til 8 stk" | Phase 2 — or Phase 1 if write tools are enabled |

## User Scenarios

### Scenario A — Quick stock check after a busy weekend

It is Monday morning in mid-April. Moohsen filters by "Lav beholdning" — six products appear in amber. Nokian Wetproof 205/55R16 and Continental EcoContact 6 195/65R15 both dropped to 3 units over the weekend. He clicks the Nokian product, finds the variant table, and types "8" into the stock field for the 205/55R16 variant to log the new shipment that arrived Friday. He saves. The amber turns green. Four products remain amber — he leaves those for the supplier call this afternoon.

### Scenario B — Customer asks about a dimension the shop may not carry

A customer emailed asking about Bridgestone Turanza T005 245/45R18. Moohsen types "Bridgestone Turanza" in search. The product appears — variants table shows 205/55R16, 215/55R17, 225/45R17, 235/45R17, but no 245/45R18. He asks the agent: "Har vi noen 245/45R18 fra andre merker?" The agent returns three: Michelin Pilot Sport 5, Continental SportContact 7, and Pirelli P Zero. He replies to the customer with those options.

### Scenario C — Purchase planning before winter tire season

It is late August. Moohsen asks the agent: "Hva bør jeg bestille av vinterdekk nå? Sammenlign lagerstatus med salgstallene fra de siste tre sesongene." The agent analyses current inventory across all winter tire variants and compares against order data from October–December 2022, 2023, and 2024 per dimension and brand. It returns a purchase recommendation table:

| Product | Current stock | Avg Oct–Dec sales (3yr) | Recommended order |
|---------|--------------|------------------------|-------------------|
| Nokian Hakkapeliitta 10 205/55R16 | 4 | 87 units | 40 |
| Nokian Hakkapeliitta 10 215/60R16 | 6 | 71 units | 30 |
| Continental IceContact 3 205/55R16 | 2 | 44 units | 20 |
| Michelin X-Ice North 4 215/55R17 | 0 | 28 units | 15 — urgent |

The agent adds: "Nokian 205/55R16 er din bestselger tre år på rad — og du starter sesongen med 4 stk. Det er den høyeste risikoen akkurat nå." Moohsen prints the recommendation and calls his Nokian rep.
