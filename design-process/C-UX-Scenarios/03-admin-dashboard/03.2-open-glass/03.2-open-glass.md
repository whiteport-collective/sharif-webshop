## What this screen does

The start screen. Moohsen sees everything that matters right now — in one view, without clicking anything.

Four priority zones:

1. **New orders** — count of orders since last login, with a mini list of the most recent (customer name, tire, amount, mounting location)
2. **Stock alerts** — products below threshold, highlighted in amber/red with current quantity and reorder suggestion
3. **Escalated messages** — customer questions the storefront agent (Scenario 02) could not resolve, waiting for Moohsen's reply
4. **Patterns** — the agent surfaces interesting patterns detected in recent data: unusual spikes, geographic clusters, seasonal shifts, repeat customers, emerging demand for a dimension not in stock

Below the priority zones: **Sales over time** — revenue chart comparing the current period to the same period 1, 2, and 3 years prior. Immediately tells Moohsen whether this season is better or worse than historical baseline.

The agent sidebar is open by default on first load, showing the morning briefing. It can be closed to gain more horizontal space.

This is the **open glass** concept: total store visibility at a glance, as if the walls of the operation are made of glass.

## User actions

| Action | Result |
|--------|--------|
| Click an order row | Opens order detail panel (or navigates to 03.3 Orders with that order selected) |
| Click a stock alert | Navigates to 03.4 Products with that product highlighted |
| Click an escalated message | Opens the conversation in the agent sidebar, ready to reply |
| Click a pattern card | Agent sidebar opens with a deeper explanation + suggested action |
| Change chart period | Toggle: This week / This month / This year |
| Change chart comparison | Toggle: vs last year / vs last 2 years / vs last 3 years |
| Click nav item | Navigates to the corresponding view (03.3–03.7) |
| Close agent sidebar | Dashboard expands to full width |

## Data connections

| Table / API | Purpose |
|-------------|---------|
| Medusa `Order` | New order count + mini order list + historical revenue |
| Medusa `LineItem` | Tire name, quantity per order + pattern detection (top sellers, missing dimensions) |

| Medusa `InventoryItem` + `InventoryLevel` | Current stock per product/variant |

| `sharif_settings` (local) | Low stock threshold setting |
| `support_escalations` (local, to be created) | Messages from storefront agent that need human reply |

## Sales chart

A line chart comparing revenue over time across up to 4 series:

| Series | Colour | Description |
|--------|--------|-------------|
| This period | Dark / primary | Current week/month/year revenue by day |
| 1 year ago | Medium grey | Same period last year |
| 2 years ago | Light grey | Same period 2 years ago |
| 3 years ago | Lightest grey | Same period 3 years ago |

**Period selector:** This week / This month / This year (toggle buttons above chart)
**Comparison selector:** Show/hide year-over-year lines (checkboxes or toggle)

X axis: dates. Y axis: NOK revenue. Hover tooltip shows exact value per series per date.

If fewer than 3 years of data exist, missing series are simply hidden — no error state.

Data source: `Order.created_at` + `Order.total` grouped by day, filtered by period, joined across years.

## Patterns section

The agent analyses recent order data and surfaces cards — one insight per card. Cards are generated on dashboard load and cached for the session. Each card has:

- **Icon** — category indicator
- **Headline** — the insight in one sentence
- **Supporting data** — 1–2 numbers that prove it
- **Suggested action** — what to do about it (optional)
- **Drill in** button — opens agent sidebar with a deeper breakdown

### Pattern types the agent looks for

| Pattern type | Example headline | Data used |
|-------------|-----------------|-----------|
| Demand spike | "205/55R16 orders up 40% this week" | `LineItem` dimension frequency vs prior week |
| Missing dimension | "6 customers searched 215/65R17 — not in stock" | `search_queries` log (if available) or order intent |
| Geographic cluster | "8 of today's orders are from Østfold" | `Order.shipping_address.city/postal_code` |
| Repeat customers | "3 customers have ordered 3+ times — none have accounts" | `Order` grouped by email |
| Seasonal shift | "Winter tire orders started 2 weeks earlier than last year" | `LineItem.metadata.season` + YoY comparison |
| Revenue anomaly | "Tuesday revenue was 3× the weekly average" | `Order.total` by weekday vs rolling average |
| Stock-demand mismatch | "Nokian Hakkapeliitta 9 205/55R16 almost out — 12 similar orders this week" | `InventoryLevel` + `LineItem` |
| Abandoned escalations | "2 customer questions unanswered for more than 24 hours" | `support_escalations.created_at` |

Pattern detection runs server-side via agent tool calls on login. Results are cached — not recalculated on every page load.

## Agent behaviour on this screen

The agent's morning briefing is a pre-composed summary built from live data:

> "God morgen, Moohsen. Det er **26 nye ordre** siden i går — **14% mer enn samme dag i fjor**.
> **3 produkter** er under minstebeholdning.
> **2 kundespørsmål** venter på svar fra deg.
> Jeg har også lagt merke til at **205/55R16-bestillinger er opp 40% denne uken** — du bør sjekke lagerstatus.
> Vil du at jeg tar deg gjennom dette?"

If Moohsen replies "ja" the agent walks him through: first the escalations (one by one, with suggested replies), then the stock alerts, then the top pattern.

Example follow-up queries on this screen:

| Query | Agent action |
|-------|-------------|
| "Hvordan ser salget ut sammenlignet med i fjor?" | Calls `getOrdersSummary` for current + prior years, narrates the trend |
| "Hva er det mest solgte dekket denne måneden?" | Calls `listOrders`, aggregates by `LineItem.title` |
| "Vis meg mønsteret med 205/55R16" | Expands that pattern card, calls `listOrders({ dimension: "205/55R16" })` |
| "Er det noe jeg bør gjøre nå?" | Agent prioritises: replies to escalations first, then stock reorder, then patterns |

## User Scenarios

### Scenario A — Reading the morning before opening the doors

It is 08:04. Moohsen has his coffee. The dashboard shows four cards across the top: 9 orders today, NOK 42 700 in revenue, 3 pending mounting bookings, 2 products with low stock. Below the cards, the new orders section lists the five most recent — he can see Kari Nilsen from Kongsberg ordered Nokian Hakkapeliitta 10 205/55R16 with mounting at Drammen, and someone from Fredrikstad ordered Continental WinterContact 225/45R17 for home delivery. He scrolls to the patterns section. One card reads: "215/65R16-bestillinger er opp 58% denne uken — du har 3 stk igjen på lager." He clicks it. The agent sidebar opens and says: "De siste 7 dagene er det kommet 11 bestillinger på 215/65R16. Basert på denne takten vil du gå tom innen fredag. Vil du at jeg sjekker hvilke merker vi har igjen?"

### Scenario B — Comparing this season to last year

It is the first week of October — the start of winter tire season. Moohsen toggles the sales chart to "Denne måneden." The chart shows four lines. This year tracks above the 2023 and 2024 lines but below 2022. He types in the agent sidebar: "Hva var omsetningen i oktober de siste tre årene?" The agent responds: NOK 1.2M in 2022, NOK 890K in 2023, NOK 945K in 2024 — and "Vi er nå på NOK 312K etter 8 dager i 2025 — tilsvarer ca NOK 1.17M for hele måneden hvis takten holder." Moohsen makes a mental note and opens Orders.

### Scenario C — Responding to an escalated customer question

Moohsen notices a red badge on the escalated messages section — one message from Sofie Bergstrøm in Sandvika. She wants to know whether she can fit 225/50R17 tyres on a car that originally came with 215/55R17, and whether Sharif can advise on rim width. The agent has a suggested reply ready. Moohsen reads it, adds a line about calling the shop, and clicks Send. The card disappears from the dashboard.
