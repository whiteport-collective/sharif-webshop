## What this screen does

The agent sidebar is not a screen — it is a persistent panel that lives on the right side of every admin view. It can be opened and closed. Its state (open/closed) is remembered per session via `localStorage`.

It behaves like a trusted ops assistant: it knows what Moohsen is looking at, logs every tab switch, and answers questions about the current data context without him having to specify where he is.

This page documents the sidebar as a standalone UX component, including all its states.

---

## Sidebar states

| State | Description |
|-------|-------------|
| Closed | A floating toggle button (bottom-right) is the only visible element |
| Open — empty | First open, no messages yet. Shows a prompt: "Hva lurer du på?" |
| Open — morning briefing | On login, pre-loaded with the agent's good morning summary |
| Open — active conversation | Chat history visible, textarea active |
| Open — context switch | Subtle separator line between messages: "— Byttet til Ordre-visning —" |
| Open — escalation | A customer message card appears, with suggested reply and "Send" button |

---

## User actions

| Action | Result |
|--------|--------|
| Click toggle button | Opens/closes sidebar with slide animation |
| Type message + send | Agent responds, streams back |
| Tab switch (any nav item) | Context injected silently; separator line appears in chat |
| Click escalated message card | Expands full conversation thread |
| Edit and send reply to escalation | Sends reply back to customer via support chat |
| Scroll chat history | Normal scroll, newest at bottom |

## Context injection per view

When Moohsen navigates, the sidebar automatically appends a context note (not typed by the user):

| View | Context injected to agent |
|------|--------------------------|

| 03.3 Orders | `"User is viewing the Orders list."` |
| 03.4 Products | `"User is viewing Products & Inventory."` |
| 03.5 Customers | `"User is viewing the Customers list."` |
| 03.7 Settings | `"User is viewing Settings."` |

Visible in the chat as a grey separator line (not a message bubble):
> — Byttet til Ordre-visning —

## Morning briefing format

Shown on first load after login. Agent composes from live data:

```
God morgen, Moohsen.

Det er 26 nye ordre siden i går.
3 produkter er under minstebeholdning: Nokian Hakkapeliitta 9 (205/55R16: 2 stk), ...
2 kundespørsmål venter på svar fra deg.

Vil du at jeg tar deg gjennom dem?
```

## Escalation card format

When a storefront customer asked a question the storefront agent couldn't answer, it escalates to this sidebar. Each escalation appears as a card:

```
┌──────────────────────────────────────┐
│ 👤 Ola Nordmann — ola@example.com    │
│ "Kan jeg bytte til 215/55R16 i stedet │
│  for 205/55R16 på min bestilling?"    │
│                                       │
│ [Foreslått svar]                      │
│ "Hei Ola! Ja, det er mulig å bytte   │
│  dimensjon. Ta kontakt med oss på..." │
│                                       │
│ [Rediger]          [Send svar]        │
└──────────────────────────────────────┘
```

## User Scenarios

### Scenario A — Walking through the morning briefing

Moohsen logs in at 08:00 on a Wednesday. The sidebar is open with the agent's briefing: 19 new orders, 1 product below threshold, 3 escalated customer questions. He types "ja, gå gjennom dem." The agent starts with the three questions — presenting each as a card with the customer's message and a suggested reply. He approves two unchanged and edits the third — a question about fitting winter tires without changing rims on a Tesla Model Y — adding a note to call the shop. He clicks Send on each. The agent moves to the stock alert: Nokian Hakkapeliitta 9 205/55R16, 2 units remaining. He asks: "Hvem er min Nokian-kontakt?" The agent does not have that data and says so clearly. He writes the rep's name on a Post-it and moves on.

### Scenario B — Context switch mid-investigation

Moohsen is in the Orders view looking at a spike in Yokohama BluEarth-Es ES32 185/65R15 orders. He clicks Products in the nav. The sidebar shows a grey separator: "— Byttet til Produkt-visning —" He types: "Vis meg lagerstatus for Yokohama BluEarth-Es 185/65R15." The agent responds: 7 units in stock — and adds: "Basert på denne ukens bestillingsrate går du tom på ca 4 dager." He filters the product list to highlight that row and makes a note to reorder.

### Scenario C — Responding to an escalation at 14:30

A new escalation appeared while Moohsen was with a customer. Henrik Paulsen ordered Pirelli Scorpion Winter 235/55R19 and booked mounting at Fjellhamar on Thursday — he wants to know if aftermarket alloy rims take longer to mount. The storefront agent flagged it as outside its knowledge. The admin agent has drafted a reply: experienced with aftermarket rims, may take 30 minutes longer, mention it on arrival. Moohsen adds: "Du kan ringe oss på 32 88 XX XX dagen før om du vil bekrefte" — and sends it. Henrik's question is resolved.
