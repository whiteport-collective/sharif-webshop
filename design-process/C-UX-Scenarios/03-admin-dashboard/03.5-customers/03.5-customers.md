## What this screen does

Customer directory with order history and contact details. Moohsen can find a customer by name or email, see everything they have ordered, and check if they have any open support conversations. This is also the screen the agent uses when Moohsen asks "find me the customer who ordered last Tuesday and didn't get a booking confirmation."

Note: many Sharif customers are **guests** (no account) — they are still tracked via order data. The customer view covers both registered accounts and guest orders grouped by email.

---

## User actions

| Action | Result |
|--------|--------|
| Search by name / email / phone | Filters customer list in real time |
| Filter by has account / guest | Toggle |
| Filter by order count | e.g. "more than 2 orders" |
| Filter by last order date | Date range |
| Click customer row | Opens customer detail panel |
| Click order in detail panel | Opens that order in 03.3 Orders |
| Ask agent "find customers who ordered twice or more" | Agent calls `listCustomers({ min_orders: 2 })` |

## Customer list columns

| Column | Source | Notes |
|--------|--------|-------|
| Name | `Customer.first_name + last_name` | Or "Guest" if no account |

| Phone | `Customer.phone` | From order shipping address if guest |
| Orders | Count of `Order` linked to customer | |

## Customer detail panel (right column)

| Section | Fields | Source |
|---------|--------|--------|
| Header | Name, email, phone, type badge | `Customer` |
| Address | Default shipping address | `Customer.shipping_addresses` |
| Distance | km from Drammen | `calculateDistance(address, "Drammen")` |
| Order history | List: order ID, date, tires, amount, status | `Order[]` linked to customer |
| Support history | Open/closed conversations from storefront chat | `support_escalations` filtered by email |
| Notes | Internal notes (editable) | `Customer.metadata.admin_notes` |

## Data connections

| Table / API | Purpose |

---

## Guest customer handling

Guests have no `Customer` record. They are grouped by email from `Order.email`. When Moohsen searches for a guest, the system queries orders by email and constructs a virtual profile from the order data.

---

## Agent behaviour on this screen

Context injected: `"User switched to Customers view."`

Example queries:

| Query | Agent action |
|-------|-------------|
| "Finn Harriet Andersen" | Calls `listCustomers({ search: "Harriet Andersen" })` |
| "Vis kunder som bor mer enn 100km fra Drammen" | Calls `listCustomers()` + `calculateDistance()` per address |
| "Hvem har bestilt flest ganger?" | Calls `listCustomers()`, sorts by order count |
| "Har denne kunden åpne spørsmål?" | Calls `getEscalatedMessages({ email })` |
| "Send en oppfølgingsmelding til Erik Hansen" | Phase 2 — read-only in Phase 1 |

## User Scenarios

### Scenario A — A loyal customer calls and wants his history

Rolf Stranden from Lierbyen calls on a Tuesday. He ordered from Sharif before and wants to know what tires he put on his Volvo last winter. Moohsen types "Rolf Stranden" — three orders, NOK 18 200 total, last order November 2024: Nokian Hakkapeliitta 10 SUV 235/65R17 × 4, mounting at Drammen, completed. Moohsen reads it back to Rolf. Rolf confirms — that is exactly what he wants. Moohsen checks stock and says he will follow up by email in ten minutes.

### Scenario B — Finding repeat guests who have no account

The dashboard surfaced a pattern: several customers have ordered 3+ times but have no registered accounts. Moohsen asks the agent: "Vis kunder uten konto som har bestilt mer enn to ganger." The agent returns 8 guest customers. Top result: Anette Svensson from Drammen, four orders since 2022, NOK 24 400 total, always picks mounting at Drammen. He notes her email — he will invite her to create an account for faster checkout next time.

### Scenario C — Tracing a complaint

Lars Brekke from Stavanger left a message saying a tire arrived damaged. Moohsen searches for Lars Brekke — one order: #1034, Goodyear EfficientGrip Performance 2 225/45R17 × 2, home delivery, completed 12 days ago. He asks the agent: "Har denne kunden sendt noe til kundestøtten?" The agent checks escalations — nothing. Moohsen adds an internal note: "Kunde meldte skadet dekk, se opp for returforespørsel" — and passes the case to his assistant.
