## Overview

**Page Purpose:** Celebrate the purchase and let Harriet book a mounting time. This is the reward step -- payment is done, now she picks a time. The tone shifts from functional to celebratory.

**User Situation:** Harriet just completed payment via Klarna/Qliro and has been redirected back to Sharif. She sees her order is confirmed and can immediately book a mounting time at her selected shop.

**Success Criteria:** Harriet books a mounting time (or chooses to do it later via email reminder). She leaves with a screenshot-friendly confirmation.

**Entry Points:**
- Redirect from 01.5-Payment (success URL with order reference)

**Exit Points:**
- Booking confirmed -> confirmation state shown (scenario complete)
- "Choose time later" -> email reminder sent, confirmation shown (scenario complete)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Redirect from payment | Confirmation shown, calendar loading | Select date |
| Calendar Loaded | Slots fetched | Available dates highlighted, time slots shown | Tap date, tap slot |
| Slot Selected | User taps a time | Selected slot highlighted (brand-primary), CTA enabled | Confirm or change |
| Booking Loading | CTA tapped | CTA shows spinner, slots disabled | Wait |

| Skip Chosen | Skip link tapped | Calendar section replaced by skip confirmation, email triggered | Close |
| Calendar Empty | No slots available | "Ingen ledige tider denne uken" message, skip link prominent | Choose later or call |
| Error | Booking API fails | Inline error, slots remain selectable | Retry |

## Technical Notes

- Sharif brand surface returns here after the payment provider's UI. The retro racing aesthetic is back.
- Google Calendar API integration for POC -- fetch available slots from shop calendar.
- Saturday slots should be prominent in the calendar -- that is when Harriet (and most consumers) are free.
- The confirmation state must be screenshot-friendly -- date, time, full address, all visible without scrolling.
- Email reminder for "choose time later" should include a direct booking link with the order reference.
- The confirmation icon animation should feel like a small win -- satisfying but not over-the-top.
- Order reference from payment redirect URL is required to display correct order data.
- Consider pre-selecting the nearest Saturday with available slots when the calendar loads.
