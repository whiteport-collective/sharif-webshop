## Overview

**Page Purpose:** Complete the entire purchase in one continuous panel вҖ” choose delivery, fill in contact/address, pay, book mounting, see confirmation.

**Architecture:** The checkout is a 4th panel in the FlowShell system. It slides up from below over the results view (which stays rendered underneath). No page navigation occurs вҖ” only React state transitions between steps. The URL does NOT change.

**User Situation:** Harriet tapped "Velg disse" on a product card. The tire was added to cart (if not already there), then the checkout panel slid up. She sees the delivery options immediately.

**Success Criteria:**
- She picks Fjellhamar and fills in her info without leaving the flow
- Payment completes without any page redirect
- Booking time appears and she picks a slot
- Confirmation screen scrolls in вҖ” she knows it's done

**Entry Points:**
- Tap "Velg disse" on 01.2-Product Cards or 01.3-Product Detail вҶ’ adds to cart вҶ’ panel slides up
- Tap buy button when tire already in cart вҶ’ panel slides up (no duplicate added)

**Exit Points (within panel):**
- Back button / scroll-up gesture вҶ’ navigate back through steps (or close panel вҶ’ back to results)
- Order placed вҶ’ scroll down to inline confirmation (back nav locked from there)

## Architecture

### FlowShell Panel Stack

```
z: 60 вҖ” Fixed service buttons (language, cart badge) вҖ” always visible
z: 30+вҖ” CHECKOUT PANEL вҶҗ active when view === "checkout"
z: 20 вҖ” QUANTITY-SHOP PANEL
z: 10 вҖ” RESULTS PANEL вҶҗ pushed off-top (-translate-y-full) when checkout is open
z: 0  вҖ” HOME PANEL
```

All panels use `absolute inset-0`, `overflow-y-auto`, `transition-transform duration-500`.

Checkout panel: `translate-y-0` when active, `translate-y-full` when inactive.

### Step State Machine

Steps are managed with React state (`step: string`) inside `CheckoutPanelContent`. No URL params involved.

```
"delivery" вҶ’ "address" вҶ’ "payment" вҶ’ "booking" (workshop only) вҶ’ [order placed]
```

Back navigation:
```
"delivery" вҶ’ onBack() вҶ’ setView("results")
"address"  вҶ’ setStep("delivery")
"payment"  вҶ’ setStep("address")
"booking"  вҶ’ setStep("payment")
```

### Back Gesture

- **Scroll-up at top of panel** (wheel deltaY < вҲ’40): triggers back navigation with 600ms lockout to prevent double-fire
- **Touch swipe down** (> 60px from top): same

## Loading State

When the checkout panel opens, it fetches cart, customer, shipping methods, and payment methods in parallel. While loading, a **skeleton** is shown that mirrors the real layout:

- Three bordered radio rows (delivery options skeleton)
- A grid of input field placeholders (address form skeleton)
- A dimmed payment section heading

No spinner. The skeleton has `animate-pulse` shimmer.

5. On payment confirmed вҶ’ `onSuccess(orderId)` вҶ’ inline confirmation

## Technical Notes

- **No page navigation:** The panel uses `setView("checkout")` in FlowShell, not `router.push`. URL remains unchanged.
- **Cart idempotency:** Before calling `addToCart`, FlowShell checks `cart.items` for an existing line with the same variant ID. If found, skips add.
- **Instant panel open:** The checkout panel opens immediately on "Velg disse" вҖ” it does not wait for `addToCart` to complete. A `cartLoading` prop is passed from FlowShell; while true, `CheckoutPanelContent` shows skeleton and holds its cart fetch. When `cartLoading` becomes false (addToCart done), the fetch runs. This eliminates perceived delay while avoiding a race condition.
- **Fresh cart on open:** `CheckoutPanelContent` is keyed with `checkoutKey` вҖ” incremented each time checkout opens вҖ” ensuring a fresh mount and fresh data fetch each time.
- **Stripe Elements:** `PaymentWrapper` initializes Stripe with `NEXT_PUBLIC_STRIPE_KEY`. Stripe Elements are client-side only.
- **Server actions:** `setAddressesInPanel`, `placeOrderInPanel`, `saveBookingToCart` are server actions that return values instead of calling `redirect()`.
- **Scroll-up lockout:** Back gesture uses a 600ms lockout ref (`backLocked`) to prevent double-fire on trackpad momentum scroll.
