## Overview

**Page Purpose:** Let Harriet pick how many tires and where to get them mounted. Two quick decisions, then she pays. This is the last step before the payment hand-off.

**User Situation:** Harriet tapped "Jeg tar disse" on the product detail overlay (01.3). The overlay has transitioned into this view — the product she selected is now a compact summary at the top. She has committed to the tire. All that remains is quantity and location.

**Success Criteria:**
- Harriet selects quantity (2 or 4) within 3 seconds — the default of 4 handles most cases
- She picks a mounting shop within 5 seconds — two clear options, no decision paralysis
- The live total on the CTA button gives her confidence — no hidden costs
- She taps "Betal nå" within 15 seconds of arriving on this view

**Entry Points:**
- Tap "Jeg tar disse" / "I'll take these" CTA on 01.3-Product Detail

**Exit Points:**
- Tap "Betal nå — 1 996 kr" / "Pay now — 1 996 kr" CTA → 01.5-Payment
- Tap back / browser back → return to 01.3-Product Detail

## Technical Notes

- **Parallax continuity:** This view is part of the continuous scrolling surface. The transition from 01.3 should feel like the product detail overlay collapses into the compact summary bar and the quantity/shop content scrolls into place. No hard page break.
- **Live price calculation:** Total = quantity x unit price. Calculated client-side. The CTA label, summary line, and summary total all update simultaneously when quantity changes. Use a reactive binding — no API call needed for price updates.
- **State carried from 01.3:** Product data (name, image, price, dimension) is passed from the product detail view. This can be URL params, session state, or a shared store. No separate API fetch should be needed.
- **Smart defaults:** Quantity defaults to 4 (most common tire purchase). Shop defaults to Fjellhamar (first in list). The page is immediately actionable — Harriet can tap the CTA without changing anything.
- **Availability hints:** The "Mange ledige tider denne uken" / "Plenty of slots this week" text is a soft confidence signal. It does not commit to specific time slots — booking happens after payment (01.6). This can be a static string for POC, or driven by a simple API flag (available / limited / full) in production.
- **Sticky CTA:** Positioned sticky at bottom of viewport, not within scrollable content. Must account for mobile safe area (home indicator). Content area has bottom padding (`qty-v-space-2xl`) so the last element is not hidden behind the CTA.
- **Toggle button behavior:** Only one quantity option can be selected at a time. Only one shop can be selected at a time. Both use `role="radiogroup"` with `role="radio"` on individual options. Arrow keys navigate between options.
- **Accessibility:** Quantity toggles and shop cards are keyboard-navigable. Selected state communicated via `aria-checked="true"`. Live price updates announced via `aria-live="polite"` region on the order summary. CTA button has descriptive label including the total amount.
- **Back navigation:** Browser back returns to 01.3-Product Detail with previous state preserved. The product detail overlay should re-open.
- **Number formatting:** Norwegian locale uses space as thousands separator (1 996 kr). English locale uses comma (1,996 NOK). Apply locale-aware formatting.
