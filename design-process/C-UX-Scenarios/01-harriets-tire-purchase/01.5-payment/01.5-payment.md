## Overview

**Page Purpose:** Harriet completes her purchase via Stripe. Unlike the old design (Klarna full redirect), this is a Sharif-owned page that embeds the Stripe Elements payment form. The visual transition from the rest of the flow is seamless — Sharif branding persists.

**User Situation:** Harriet tapped "Betal nå — 1 996 kr" on the quantity-and-shop view. She arrives at the checkout page which already shows her order summary and an embedded card form.

**Success Criteria:** Harriet enters her card details, pays, and is redirected to the booking view (01.6).

**Entry Points:**
- CTA "Betal nå" on 01.4-Quantity & Shop → navigates to `/no/checkout`

**Exit Points:**
- Payment complete → redirect to `/no/order/confirmed?cart_id={id}` → 01.6-Book Mounting
- Payment failed → error state on same page, retry option
- Back button → return to 01.4-Quantity & Shop

## Technical Notes

- **Route:** `/no/checkout`
- **Backend:** Medusa v2 handles cart creation and Stripe PaymentIntent initialization
- **Payment flow:** Cart → add line item → add email → initiate payment collection → create Stripe payment session → render PaymentIntent client secret in Stripe Elements → complete cart after payment
- **Stripe mode:** Test Mode (`sk_test_...` / `pk_test_...`)
- **Test card:** `4242 4242 4242 4242`, any future expiry, any CVC
- **No redirect:** The payment form is embedded. Harriet never leaves Sharif's domain.
- Order webhook from Stripe → Medusa marks order as paid → order visible in Medusa Admin
