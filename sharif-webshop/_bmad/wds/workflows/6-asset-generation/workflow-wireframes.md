## Drawing Conventions

These rules apply to ALL wireframes generated by agents.

### Text Representation
- **Headlines:** Real text from specs — the wireframe should read like a real page
- **Body text:** Gray placeholder bars (rectangles), one per line:
  - Color: `#ced4da`, opacity: `18`, height: `10`
  - Stack with ~20px spacing, last line at 60-70% width
  - Group each paragraph's bars (e.g. `body-para-1`)
- **Never** use text elements for body copy — always gray bars

### Buttons
- Always group button rectangle + button text with a shared `btn-*` groupId
- Buttons use `roundness: { "type": 3 }`, body text bars do not

### Image Placeholders
- Cross-hatch fill: `fillStyle: "cross-hatch"`, `backgroundColor: "#d0ebff"`, `strokeColor: "#868e96"`

### Draft Versioning
- Never edit a wireframe file in-place between review rounds
- Copy previous draft to a new file: `draft-01` → `draft-02` → ... → final (no suffix)

### No Annotations
- Wireframes show only what the user sees — no SECTION labels or dimension markers
